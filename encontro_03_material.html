1 Entendendo os métodos HTTP

Toda requisição HTTP carrega junto um método — uma palavra que indica o que o cliente quer fazer. Quando você digita um endereço no navegador e pressiona Enter, o navegador envia uma requisição com o método GET. Mas existem outros métodos para outras intenções.

GET
Buscar informações. Não altera nada no servidor.
POST
Criar algo novo. Envia dados no corpo da requisição.
PUT
Atualizar algo que já existe. Substitui o recurso inteiro.
DELETE
Apagar algo. Identifica o que deve ser removido.
Por que isso importa? Imagine uma API de agendamentos. O endereço /agendamentos pode fazer coisas completamente diferentes dependendo do método: GET /agendamentos → lista todos os agendamentos; POST /agendamentos → cria um novo agendamento; PUT /agendamentos → atualiza um agendamento; DELETE /agendamentos → remove um agendamento. O mesmo endereço, quatro comportamentos distintos. Isso é o design de uma API REST — padrão usado em todo o mercado.
GET vs POST — a diferença mais importante agora O método GET não carrega dados no corpo da requisição — apenas na URL. O método POST carrega dados no corpo (body) da requisição, de forma que não aparecem na URL. Por isso usamos POST quando queremos enviar informações para criar algo: nome, e-mail, senha, título — qualquer dado que o servidor precisa receber para criar um novo registro.

2 Postman — sua ferramenta de testes de API

Por que precisamos de uma ferramenta de teste? O navegador só consegue enviar requisições GET facilmente (quando você digita um endereço). Para testar rotas POST — que recebem dados no corpo da requisição — o navegador não serve. Profissionais usam ferramentas específicas para isso. O Postman é a mais utilizada no mercado: permite criar e enviar qualquer tipo de requisição, configurar headers e body, e ver a resposta completa com status code. Usaremos a versão web, que funciona direto no navegador ao lado do Codespace — sem instalação. É o que usaremos do Encontro 03 até o fim da UC.

2
Entender a interface do Postman
Conhecer a ferramenta antes de usar

Para criar uma nova requisição, clique no botão New (canto superior esquerdo) → HTTP Request. Uma aba de requisição se abre com as seguintes partes principais:

  • Método HTTP — o seletor à esquerda da URL (GET, POST, PUT, DELETE). Clique nele para trocar o método.
  • URL — o campo onde você digita o endereço completo da requisição, incluindo o caminho da rota.
  • Params — aba para adicionar query params (?chave=valor) de forma visual. O Postman monta a URL automaticamente.
  • Body — aba para escrever os dados enviados no corpo (POST e PUT). Selecione raw e depois escolha JSON no menu dropdown que aparece à direita.

Quando você clica em Send, o Postman envia a requisição e mostra a resposta na parte inferior — incluindo o status code, o tempo de resposta e os dados retornados.

Como obter a URL do Codespace: No VS Code, vá na aba Ports no painel inferior, passe o mouse sobre a porta 3000 — um ícone de globo aparece. Clique nele para abrir no navegador e copie a URL da barra de endereços. Cole essa URL no Postman e adicione o caminho da rota no final.

3 Manual — API de Frases do Dia

Vamos construir uma pequena API para gerenciar frases do dia. Ela vai ter uma rota GET (para listar as frases) e uma rota POST (para adicionar uma nova). Ao final, você vai adaptar tudo isso para o seu projeto.

3
Preparar o index.js para o exercício
Começar com uma base limpa para o exemplo

Para este exercício, vamos trabalhar em um arquivo separado para não misturar com o projeto. Crie um arquivo chamado frases.js na raiz do projeto.

Vamos construir esse arquivo passo a passo. Por enquanto, escreva apenas a estrutura inicial — que você já conhece do encontro anterior:

const express = require('express')
const app = express()

const PORT = process.env.PORT || 3000

Salve o arquivo. Não inicie o servidor ainda — vamos adicionar mais coisas primeiro.

4
Adicionar o express.json() — necessário para o POST funcionar
Uma linha que habilita o servidor a ler dados enviados no corpo da requisição

Quando um cliente envia dados em uma requisição POST, esses dados chegam ao servidor como um texto bruto. O servidor precisa interpretar esse texto e transformá-lo em um objeto JavaScript que o código consegue usar.

O express.json() faz exatamente isso: ele intercepta as requisições que chegam, lê o corpo em formato JSON e coloca o resultado em req.body — que você vai usar nas rotas POST.

Adicione esta linha após a criação do app e antes das rotas. A ordem importa: ela precisa estar antes das rotas para que funcione em todas elas.

const express = require('express')
const app = express()

const PORT = process.env.PORT || 3000

app.use(express.json()) // habilita leitura do corpo das requisições
Se esquecer essa linha: O req.body nas rotas POST vai ser undefined. Se seus dados chegarem como undefined, verifique primeiro se essa linha está no código.
5
Criar os dados iniciais em memória
Por enquanto os dados ficam em um array no código — o banco de dados virá mais adiante

Antes de criar as rotas, precisamos de onde os dados vão viver. Por enquanto vamos usar um array simples declarado no código. Isso significa que os dados existem enquanto o servidor está rodando — se você reiniciar o servidor, voltam para o estado inicial. Isso é esperado neste ponto da UC. Nos próximos blocos você vai migrar para um banco de dados real.

Adicione o array após o app.use(express.json()):

const frases = [
  { id: 1, texto: 'A jornada de mil milhas começa com um único passo.', autor: 'Lao-Tzu' },
  { id: 2, texto: 'A simplicidade é a sofisticação máxima.', autor: 'Leonardo da Vinci' }
]
Cada frase é um objeto com três propriedades: id (número único), texto e autor. Essa estrutura é a mesma que vai para o banco de dados — você está praticando o modelo de dados do seu projeto.
6
Criar a rota GET /frases
Retorna todas as frases cadastradas

A rota GET é a mais simples: o cliente pede, o servidor responde com os dados. Neste caso, respondemos com o array inteiro de frases.

app.get('/frases', (req, res) => {
  res.json(frases)
})

Agora adicione o app.listen ao final do arquivo para poder testar:

app.listen(PORT, () => {
  console.log(`Servidor rodando em http://localhost:${PORT}`)
})

Agora inicie este servidor no terminal. Como o index.js principal pode já estar rodando, abra um novo terminal (+ no painel de terminais) e rode:

node frases.js

8
Criar a rota POST /frases
Recebe dados do cliente e adiciona uma nova frase ao array

A rota POST é onde as coisas ficam mais interessantes: o cliente envia dados (texto e autor) e o servidor cria um novo registro.

Veja como ler os dados enviados pelo cliente: eles chegam dentro de req.body como um objeto JavaScript — graças ao express.json() que adicionamos no Passo 4. Usamos desestruturação para extrair as propriedades que precisamos.

Adicione esta rota antes do app.listen:

app.post('/frases', (req, res) => {
  // Extrair os dados enviados pelo cliente do corpo da requisição
  const { texto, autor } = req.body

  // Criar o novo objeto com um ID único
  const novaFrase = {
    id: frases.length + 1,
    texto,
    autor
  }

  // Adicionar ao array e retornar o novo item criado
  frases.push(novaFrase)
  res.status(201).json(novaFrase)
})
Por que status 201 e não 200? O status 200 significa "OK — a requisição foi bem-sucedida". O status 201 significa "Created — um novo recurso foi criado com sucesso". Usar o status correto é boa prática: quem consome a API entende exatamente o que aconteceu sem precisar ler a documentação.

Salve o arquivo. O nodemon vai reiniciar o servidor automaticamente.

🚀 Aplique no seu Projeto

Agora é hora de aplicar o que aprendeu no index.js do seu projeto. Você vai adicionar o express.json(), criar uma rota GET e uma rota POST para o recurso principal do seu projeto.

1 Adicione o express.json() no seu index.js
Abra o index.js do seu projeto e adicione app.use(express.json()) logo após a criação do app, antes de qualquer rota.
2 Crie um array com 2-3 itens de exemplo do seu projeto
Pense nos dados que seu recurso principal precisa ter. Exemplo: se é um sistema de agendamentos, cada item pode ter id, cliente, servico, horario. Crie o array logo após o express.json().
3 Crie a rota GET para listar os itens
Substitua a rota GET que retornava dados estáticos por uma que retorna o array que você criou. O caminho deve ser o nome do recurso no plural. Exemplos: /agendamentos, /produtos, /tarefas.
4 Crie a rota POST para adicionar um novo item
Siga o mesmo padrão do exemplo: leia os dados de req.body, crie o novo objeto com ID automático, adicione ao array com push e responda com status 201 e o item criado.
5 Teste as duas rotas no Postman
GET deve retornar a lista. POST com os dados corretos no body (raw → JSON) deve retornar o item criado com status 201. Em seguida, faça um GET novamente para confirmar que o item foi adicionado ao array.
⭐ Quer ir além? (opcional)
  • Trate o caso de body vazio: O que acontece se alguém fizer um POST sem enviar dados? Adicione uma verificação: se texto ou autor não existirem no body, retorne status 400 com a mensagem { "erro": "texto e autor são obrigatórios" }.
  • Salve suas requisições no Postman: Clique em Save em cada requisição e crie uma Collection com o nome do seu projeto. Isso evita redigitar a URL toda vez — nas próximas aulas suas requisições estarão esperando por você.
