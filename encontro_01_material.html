1 O que você vai construir nesta UC

Um sistema completo, do zero Você vai construir um backend — o servidor que processa dados e regras de negócio — com um banco de dados real, e um frontend simples para usar o sistema pelo navegador. No caminho, vai usar as mesmas ferramentas que profissionais usam em empresas de verdade: GitHub, VS Code, Postman, e ao final, deploy em produção.
Como funciona cada encontro Toda aula segue o mesmo formato: uma explicação rápida do que vamos construir, um manual passo a passo com tudo explicado, e tempo para aplicar diretamente no seu projeto. Você não vai fazer exercícios soltos sem sentido — quase tudo que você aprende entra direto no seu sistema.
Você não precisa saber tudo hoje A escolha de hoje é sobre o que o sistema faz, não sobre como construir. A parte técnica vem encontro a encontro, sempre explicada do zero antes de ser cobrada.

Requisitos mínimos de qualquer projeto: CRUD com banco de dados real · sistema de login com JWT · frontend com pelo menos 2 telas · consumo de ao menos uma API externa · deploy no final · README no GitHub.

3 O que é MVP — e por que começar pequeno

MVP = Produto Mínimo Viável É a versão mais simples do seu sistema que já resolve o problema central. Não é a versão "incompleta" — é a versão essencial. Tudo que não for indispensável fica de fora por enquanto e pode ser adicionado depois, se houver tempo.

Exemplo com o tema Biblioteca Digital:

✓ Está no MVP
  • Cadastrar um livro
  • Listar livros disponíveis
  • Registrar um empréstimo
  • Login simples
✗ Fica para depois
  • Notificação de atraso por e-mail
  • Relatórios de acervo
  • Reserva antecipada de livros
  • Avaliação e resenha de livros
Por que isso importa Ao longo da UC, vai surgir vontade de adicionar mais coisas. Use o MVP como âncora: se a ideia não está nele, anote para depois e continue focado. Projetos que crescem sem controle raramente são concluídos a tempo.
